ASU Transcripts

=

Project Goal

The goal of this project was to transform the manual insertion process of transcript data into an automated, efficient system for Arizona State University (ASU) Admissions. We aimed to streamline the critical data processing pipeline, reduce manual errors, and significantly cut down processing time, ultimately improving the overall efficiency of student admissions handling.

Project Background

This project focused on reducing the time and effort involved in extracting and storing critical transcript information for ASU Admissions. To achieve this, we implemented a PostgreSQL database for storing transcript data, which replaced the existing manual data entry process. Using AWS technologies, including S3 Buckets for data storage and AWS Textract for intelligent document processing, we were able to automate the extraction of transcript information and format it into a CSV file. This innovation resulted in an 80.94% reduction in data extraction time, enhancing the speed of processing transcripts for incoming students. The project team adopted a Scrum approach to efficiently manage the project, utilizing Taiga for sprint planning, backlog management, and overall project tracking. The systematic approach not only ensured timely deliveries but also maintained a high level of organization and accountability within the team. Our project efforts led to securing grants totaling $5,000 through pitch competitions, highlighting our solution's impact and value in streamlining a critical administrative function at ASU.
